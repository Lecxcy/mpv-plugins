# enhanced-ab-loop 手动测试指南

本文档是给你在真实 mpv 里手动过一遍功能用的检查清单，对应 [SPEC.md](SPEC.md)
里定下来的每一条行为。IPC 自动化测试已经跑过一轮（见 [README.md](README.md)
"验证"一节），这里补的是需要肉眼判断观感（是否流畅、是否卡顿、OSD 文字是否
符合预期）的部分，以及自动化测试没有覆盖到的交互细节。

## 准备工作

### 编译并收集成可直接使用的配置目录

```sh
cmake -S . -B build
cmake --build build
scripts/collect-dist.sh
```

`dist/` 现在是一份完整的 mpv 配置目录，`enhanced-ab-loop.so` 已经加进
`dist/mpv.conf` 的 `scripts-append` 里，`dist/input.conf` 已经带上了本插件
全部按键绑定。用它启动：

```sh
mpv --config-dir="$(pwd)/dist" <随便一个测试视频>
```

建议准备一个 **10 秒以上、有音频轨** 的测试视频（时长太短不方便操作，纯静音
文件测不出音频滤镜是否生效）。如果没有现成素材，可以用 ffmpeg 现造一个：

```sh
ffmpeg -f lavfi -i "testsrc=duration=20:size=640x360:rate=30" \
       -f lavfi -i "sine=frequency=440:duration=20" \
       -shortest test.mp4
```

### 确认插件确实加载成功

打开视频后按 `\`（或 `、`）触发 `show-state`，应该看到一行形如
`State | 00:00/00:20` 的 OSD 文字。如果完全没有反应，先检查：

- mpv 启动时命令行/日志有没有报错（找 `enhanced-ab-loop` 或
  `Cannot open`/`dlopen` 关键字）；
- `dist/plugins/enhanced-ab-loop/enhanced-ab-loop.so` 文件是否存在。

## 快捷键对照表

| 按键 | 动作 | 对应 SPEC |
| --- | --- | --- |
| `[` / `]` | 设置 A / B（插入新区间，或移动光标所在区间的边界） | §3.1 §3.2 |
| `{` / `}` | 撤销 A / B | §3.4 |
| `\` 或 `、` | 显示当前状态 OSD | — |
| `\|` | 开关循环 | §1.3 |
| `Ctrl+[` / `Ctrl+]` | 微调光标所在区间的 A 后退 / B 后退 | §3.3 |
| `meta+[` / `meta+]` | 微调光标所在区间的 A 前进 / B 前进 | §3.3 |
| `Alt+[` / `Alt+]` | 拉伸空隙前一段的终点 / 后一段的起点 | §3.2 |
| `t` | 启用/禁用光标所在区间参与循环 | §5 |
| `Ctrl+S` / `Ctrl+L` | 存档 / 读档 | §6 |
| `y` / `n` | 确认 / 取消读档时的"疑似改名"提示 | §6 |

（macOS 上 `meta` 是 `Cmd`；`Alt` 在部分终端布局下可能需要用 `option`。如果
你自己的系统按键有出入，以 `dist/input.conf` 里实际写的为准。）

## A. 插入新区间（§3.1）

- [ ] **A1 基本插入**：暂停播放，seek 到 2s 按 `[`，seek 到 5s 按 `]`。
      `show-state` 应该显示出 `[00:02.000,00:05.000]` 这样一行。
- [ ] **A2 拒绝重叠**：在 A1 的基础上，seek 到 **1s**（注意：必须在 `[2,5]`
      *之外*的空隙里，落在区间内部按 `[` 会触发 B1 的 Move A，不是这里要测
      的插入流程）按 `[`，seek 到 8s 按 `]`（新区间 `[1,8]` 整个包住已有的
      `[2,5]`，重叠）。应该看到拒绝提示（类似 `Overlap denied`），且原来的
      `[2,5]` 没有变化，也没有新增区间，待定的 A 也应该被清空（不会残留成
      "半吊子"的 pending 状态）。
- [ ] **A3 落在已有区间内部时拒绝**：seek 到 0s 按 `[`（此时 pending A =
      0），再 seek 到 3.5s（落在 `[2,5]` 内部）按 `]`。应该看到 `B denied`，
      且这次待定的 A（0s）**没有被清空**——再 seek 到 6s 按 `]` 应该能成功
      配对出一个新区间 `[0,6]`（前提是不跟其他区间冲突）。
- [ ] **A4 端点顺序反了**：seek 到 5s 按 `[`，再 seek 到 5s 之前的某个点按
      `]`。应该被拒绝，不产生新区间。

## B. Move A / Move B（§3.2）

- [ ] **B1**：光标 seek 到某个已有完整区间**内部**（比如 A1 的 `[2,5]`
      中间的 3.5s），按 `[`。应该直接把这个区间的 A 移到 3.5s（变成
      `[3.5,5]`），而不是弹出"插入新区间"的流程。
- [ ] **B2**：同理，光标在区间内部按 `]`，应该移动这个区间的 B。

## C. Extend（§3.2，新功能，重点测）

准备两个有间隔的区间，比如 `[2,5]` 和 `[10,15]`。

- [ ] **C1 extend-prev**：seek 到两段之间的空隙（比如 7s），按 `Alt+[`。
      应该把 `[2,5]` 拉伸成 `[2,7]`，`[10,15]` 不受影响。
- [ ] **C2 extend-next**：在同样的空隙位置（比如 7s，如果 C1 已经把前一段
      拉到了 7 就换一个还没被占用的空隙点，比如 8s），按 `Alt+]`。应该把
      `[10,15]` 拉伸成 `[8,15]`，前一段不受影响。
- [ ] **C3 在区间内部按 extend 应该被拒绝**：光标 seek 到某个区间内部，按
      `Alt+[`/`Alt+]`，应该都被拒绝（这是 Move 的地盘，不是 extend 的）。
- [ ] **C4 没有可拉伸的邻居时拒绝**：seek 到所有区间最前面（比如 0.5s，
      前面没有任何区间），按 `Alt+[`，应该提示"没有前一段可拉伸"一类的
      拒绝信息。
- [ ] **C5 有未完成的插入时拒绝**：按 `[` 设了一个待定 A 之后（还没按
      `]`），在空隙里按 `Alt+[`/`Alt+]`，应该被拒绝，提示需要先完成/放弃
      当前插入。

## D. Nudge（§3.3，重点测：触发条件是否够宽松）

- [ ] **D1 落在边界本身也要能触发**：seek **精确落在**某个区间的 A 点上
      （比如先用 `show-state` 看清楚 A 的精确值再 seek 过去，或者干脆循环
      跳回来之后立刻按，不要额外移动），按 `Ctrl+[`（A 后退一点）。应该
      能成功微调，不应该被拒绝——这是本次重写specifically 修的一个点：
      旧版本要求"严格在区间内部"，循环刚跳回来的落点往往正好卡在这条线上
      导致 nudge 形同虚设。
- [ ] **D2 顶到相邻区间边界时拒绝**：把两个区间紧挨得很近（比如
      `[2,5]` 和 `[5.1,8]`），光标在 `[2,5]` 内部连续按好几次 `meta+]`
      （B 前进），应该在快要碰到 `5.1` 时被拒绝，不会静默地把 B 卡到贴着
      `5.1` 的某个值——每次拒绝都应该有提示，不是没反应。

## E. 撤销端点（§3.4，重点测：不能嫁接到邻居）

准备两个区间 `[0,5]` 和 `[10,20]`。

- [ ] **E1**：光标 seek 到 `[10,20]` 内部（比如 15s），按 `{`（撤销 A）。
      `show-state` 应该显示：`[0,5]` **原样不变**，另外单独一行
      `[unset,20]`（不是把 `[0,5]` 变成 `[0,20]` 之类的东西）。
- [ ] **E2**：同理，seek 到 `[0,5]` 内部，按 `}`（撤销 B），应该看到
      `[10,20]` 原样不变，单独一行 `[0,unset]`。
- [ ] **E3 待定状态下再按会清空对应半边**：E1 之后（现在处于
      `pending.b=20` 状态），再按一次 `{`，应该提示"A not set"（因为待定
      状态里根本没有 A 可撤销）；按 `}` 才会清掉待定的 B。

## F. 核心循环机制（§2，最重要的一组，务必肉眼确认）

- [ ] **F1 单段循环**：只设一个区间，比如 `[3,6]`，**取消暂停**（空格键）
      观察播放：应该在 3s~6s 之间反复循环，画面/声音在跳转瞬间不应该有
      明显的卡顿、黑屏或者杂音（这是这次重写要解决的核心问题——原来 Lua
      版本靠 50ms 定时器轮询，跳转时机不精确）。
- [ ] **F2 多段顺序循环**：设两个区间 `[2,4]` 和 `[8,10]`，取消暂停观察：
      应该先在 `[2,4]` 里循环几圈——不对，是播完 `[2,4]` 立刻跳到
      `[8,10]` 的开头，播完 `[8,10]` 再跳回 `[2,4]` 的开头，如此往复
      （不是每段各自循环，是按位置顺序轮着播）。
- [ ] **F3 开放区间循环到文件末尾**：只按 `[` 设一个 A（比如在离文件末尾
      10 秒的地方），不设 B，也不再定义其他区间。取消暂停，让它一路播到
      文件真正结束。应该看到画面跳回这个 A 点重新播放，**不应该**跳到
      文件开头（0s）或者其他莫名其妙的位置。

## G. 循环开关（§1.3）

- [ ] **G1**：定义好任意区间后按 `|`，OSD 应提示"loop off"，取消暂停后
      播放应该正常往后播、不再自动跳转（哪怕播过了区间的 B 点）。
- [ ] **G2**：再按一次 `|`，提示"loop on"，循环恢复生效。

## H. 选段循环（§5，新功能）

- [ ] **H1**：定义三个区间 `[1,2]`、`[4,5]`、`[7,8]`，光标 seek 到中间那段
      `[4,5]` 内部按 `t`。`show-state` 里这段应该显示出禁用标记（比如
      `(off) [00:04.000,00:05.000]`）。取消暂停观察循环：应该只在
      `[1,2]` 和 `[7,8]` 之间轮播，完全跳过 `[4,5]`。
- [ ] **H2**：再对同一段按一次 `t`，应该恢复启用，重新加入循环轮换。

## I. 尾帧冻结（§4，重点测：不能显示为暂停）

- [ ] **I1**：`show-state` 看一下文件总时长（比如 20s），seek 到接近末尾
      的地方（比如 18s）按 `[`，再 seek 到文件末尾（或者干脆用 `END`/
      跳到最后）按 `]`，让这段的 B 落在真实文件末尾附近。取消暂停观察：
      播放到文件"名义"末尾时**不应该**看到暂停图标/暂停状态，画面应该
      继续像正常播放一样"多播"一小段（默认 0.3 秒），然后跳回 A 继续。
      全程盯着播放/暂停状态指示，确认没有闪现"已暂停"。
- [ ] **I2 冻结画面应该是最后一帧**：如果画面内容本身在变化（比如用真实
      视频而不是纯色测试图），观察"多播的那一小段"画面看起来应不应该是
      定格/几乎不变的（因为背后是同一帧的克隆，不是真的有新内容）。
- [ ] **I3 普通区间不应该被误加尾帧时长**：确认 A1 里 `[2,5]` 这种远离
      文件末尾的普通区间，循环节奏和总时长应该跟"文件真实内容"完全吻合，
      不会莫名其妙多播 0.3 秒。

## J. 存档 / 读档（§6）

- [ ] **J1 保存**：定义好几个区间（含至少一个禁用的），按 `Ctrl+S`，应该
      看到"Loops saved"提示。
- [ ] **J2 关闭重开**：完全退出 mpv，用同一个文件重新打开（此时区间应该
      是空的，因为文件加载会重置），按 `Ctrl+L`，应该看到"Loops loaded"，
      `show-state` 里的区间应该跟保存前完全一致（包括禁用状态）。
- [ ] **J3 没有存档时**：换一个从来没存过档的视频文件，按 `Ctrl+L`，应该
      提示"No archive found"，不应该报错或者加载出奇怪的区间。
- [ ] **J4 疑似改名确认**：把 J1/J2 用过的视频文件复制一份、改成不同的
      文件名（内容完全一致），用新文件名打开 mpv，按 `Ctrl+L`。应该弹出
      一条不带任何路径信息的提示（类似"发现内容相同但名字不同的存档"），
      按 `y` 确认加载，区间应该正确恢复；如果按 `n`，应该看到"Cancelled"，
      区间保持为空。
- [ ] **J5 确认后再次保存应该完成迁移**：J4 按 `y` 之后，再按一次
      `Ctrl+S`，然后去 `dist/loop-segments/`（或者你实际用的 config-dir 下
      的 `loop-segments/`）翻一下对应的 json 文件，确认里面只有新文件名
      对应的那条 entry，旧文件名的 entry 已经不在了（不会留孤儿）。
- [ ] **J6 存档文件里应该看不到明文路径**：随便打开 J1 生成的那个 json
      文件（`loop-segments/<一串哈希>.json`），确认打开的内容里全是哈希
      字符串，**没有**你的真实文件名、目录名或者用户名。

## K. 综合/边界情况

- [ ] **K1 极短区间**：尝试定义一个只有几十毫秒宽的区间（比如
      `[5.00,5.05]`），确认 `show-state` 能正常显示、循环也能正常在这么
      窄的范围内跳转，不应该出现旧版本"过短显示错乱"的问题。
- [ ] **K2 连续快速按键**：不暂停、正常播放中，快速连续按好几次 `[`、
      `]`、`t`、`|`（不用刻意等每次的 OSD 消失），确认 mpv 不会卡死、
      崩溃，或者进入什么奇怪的状态——最后停下来后 `show-state` 应该能
      给出一个自洽（不矛盾）的结果。
- [ ] **K3 全屏播放**：切到全屏（`ENTER`）之后重复 F1/F2 的循环测试。
      旧 Lua 版本的已知问题是"全屏播放时有概率遇到大量丢帧"，这次重写
      理论上不再依赖定时器轮询，应该会好很多，但值得实测确认一下。
- [ ] **K4 播放速度调整后循环是否还准**：按 `c`/`x`（如果你的 `input.conf`
      还留着这两个测试用的速度绑定）把倍速调到 2x 或 0.5x，确认循环
      跳转依然在正确的时间点触发（原生 pts 判断应该不受播放速度影响，
      但值得实测确认一下）。

## 遇到问题时怎么反馈

如果某一项没通过，把下面这些信息带回来，方便定位：

1. 具体是上面哪一条编号（比如 "F2 不通过"）。
2. 你按的键、seek 到的具体时间点、`show-state` 当时显示的内容。
3. mpv 启动时加的参数（尤其是有没有用 `--config-dir="$(pwd)/dist"`）。
4. 如果方便的话，加 `-v` 或 `--msg-level=all=v` 跑一遍、把相关的日志片段
   贴出来（关键字找 `ab-loop`、`enhanced-ab-loop`、`hr-seek`）。

## 已知限制（不是 bug，测到这些不用报）

- 没法把进度条/章节跳转/`seek` 命令直接拖到尾帧冻结延长出来的那一小段
  时间里（demuxer 层面这段本来就不存在）。
- 进度条/剩余时间这类基于 `duration` 属性的显示不会把冻结时长算进去，
  播完真实内容后进度条会先显示 100%，画面还会继续"播"一小会儿。
- uosc 进度条目前只会展示"当前激活的一段"（原生 `ab-loop-a`/
  `ab-loop-b`），还看不到全部多段区间——这是故意留到以后单独做的
  uosc 派生插件里解决，见 [SPEC.md](SPEC.md) §7。
- §6 存档消歧遇到"两个以上内容相同的候选"时不会弹出候选列表让你选，
  直接按"没有存档"处理，需要手动为当前文件重新存一份。
